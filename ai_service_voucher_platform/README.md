# 广西制造业服务券平台 AI 审核系统 (V2 - API对接版)

本项目是基于本地大语言模型 Qwen3 实现的 **服务产品上架合规性AI审核** 系统。

此版本包含两种运行模式：
1.  **正式API服务 (`main_api.py`)**: 部署一个符合甲方接口规范的Web服务，用于生产环境对接。
2.  **快速演示脚本 (`api_demo.py`)**: 用于内部快速验证和向非技术人员直观展示系统核心能力。

## 🚀 如何运行

### 第一步: 环境准备

```bash
# 安装所有Python依赖 (如果已安装过旧版，此命令会自动补充新依赖)
pip install -r requirements.txt
第二步: 运行Ollama大模型 (必须)
确保Ollama已安装。在一个独立的终端里执行以下命令来启动Qwen3模型服务：

ollama run qwen3:8b
此终端需要保持运行。

第三步: 运行后台“守护进程” (必须)
打开第二个终端，运行此命令。它会构建初始知识库，并持续监控 policy_source 文件夹以实现政策自动更新。

python watcher.py
此终端也需要保持运行。

第四步: 选择一种模式启动应用
现在，您可以在 第三个终端 中选择以下一种方式来运行：

模式A: 启动正式API服务 (用于与甲方对接)
运行以下命令来启动生产级Web服务器：

uvicorn main_api:app --host 0.0.0.0 --port 8000
服务将运行在 http://localhost:8000。
/servceName1/check_service_content/ 端点已准备好接收甲方的POST请求。
终端会实时打印API请求接收、后台处理和回调通知的详细日志。
模式B: 运行快速演示脚本 (用于内部展示)
如果只是想快速验证一下核心逻辑，运行：

python api_demo.py
脚本会模拟三个典型的审核场景（通过/不通过/部分通过），并直接在终端打印出清晰的审核结果。
👨‍💻 非技术人员操作指南
当您收到新的政策文件时（例如《关于调整XX的通知.pdf》），请将该文件拖拽到项目内的 policy_source 文件夹中。

watcher.py 所在的终端会立刻打印出“检测到文件变动”的日志并开始学习。几分钟后，无论是正式API还是演示脚本，其审核能力都会自动更新，无需重启任何程序。
